# Firestore インデックス設定手順

## 📋 概要

チャット機能でFirestoreを使用する際、`where` + `orderBy` のクエリを実行するために**複合インデックス**の作成が必要です。

---

## 🔧 必要なインデックス

### 1. chat_messages コレクション

| フィールド | 並び順 | クエリスコープ |
|-----------|--------|---------------|
| channel_id | 昇順 (Ascending) | Collection |
| created_at | 昇順 (Ascending) | Collection |

---

## 🚀 インデックス作成方法

### 方法1: エラーメッセージから自動作成（推奨）

1. ブラウザで `groupware.html` を開く
2. **F12キー**を押して開発者ツールを開く
3. **Console（コンソール）**タブを選択
4. チャット画面を開く
5. コンソールに表示される **エラーメッセージ内のURL** をクリック
   ```
   🔧 Firestoreインデックスが必要です。以下のリンクから作成してください:
   https://console.firebase.google.com/v1/r/project/...
   ```
6. Firebaseコンソールが開き、インデックス作成画面が表示される
7. 「**インデックスを作成**」ボタンをクリック
8. 数分待つと「構築中」→「有効」に変わる

---

### 方法2: 手動で作成

1. [Firebase Console](https://console.firebase.google.com/) にアクセス
2. プロジェクト「**sakura-net-db**」を選択
3. 左メニューから「**Firestore Database**」をクリック
4. 「**インデックス**」タブをクリック
5. 「**複合インデックスを作成**」ボタンをクリック
6. 以下の内容を入力:

   - **コレクションID**: `chat_messages`
   - **フィールド1**:
     - フィールドパス: `channel_id`
     - 並び順: 昇順
   - **フィールド2**:
     - フィールドパス: `created_at`
     - 並び順: 昇順
   - **クエリスコープ**: Collection

7. 「**作成**」ボタンをクリック
8. 構築完了まで数分待つ

---

## ✅ 動作確認

インデックス作成後、以下を確認してください:

1. ブラウザで `groupware.html` をリロード（Ctrl+F5）
2. チャット画面を開く
3. メッセージを送信してみる
4. メッセージが表示されることを確認
5. ページをリロードしてもメッセージが残っていることを確認

---

## 🐛 トラブルシューティング

### インデックスが作成できない

- Firebaseプロジェクトの権限を確認してください
- プロジェクトが正しく選択されているか確認してください

### インデックス作成後もエラーが出る

- ブラウザのキャッシュをクリアしてください（Ctrl+Shift+Delete）
- インデックスのステータスが「有効」になっているか確認してください

### メッセージが表示されない

- コンソールにエラーが出ていないか確認してください
- Firestoreのセキュリティルールを確認してください

## 📊 Firestoreセキュリティルール（参考）

チャット機能を使用する場合、以下のセキュリティルールを設定してください:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // チャットメッセージ: 認証済みユーザーのみ読み書き可能
    match /chat_messages/{messageId} {
      allow read, write: if request.auth != null;
    }

    // チャットチャンネル: 認証済みユーザーのみ読み書き可能
    match /chat_channels/{channelId} {
      allow read, write: if request.auth != null;
    }

    // 既存のルール...
  }
}
```

**注意**: 現在は認証なしでアクセスできるようになっている可能性があります。本番環境では適切な認証を実装してください。
